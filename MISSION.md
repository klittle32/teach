# Mission: The Teach Skill itself

## Why
Get a clear, accurate mental model of how this teaching skill works — what it produces, the files it keeps, and the learning theory underneath — so future use of it (for any topic) feels deliberate rather than mysterious.

## Success looks like
- Can explain the Knowledge / Skills / Wisdom split and why each needs a different approach
- Knows what each workspace file is for (MISSION, RESOURCES, GLOSSARY, learning-records, lessons, reference, NOTES)
- Can name the difference between fluency strength and storage strength, and why the skill optimises for the latter

## Constraints
- Curiosity-driven, no downstream deadline. Keep it light — one sitting.

## Out of scope
- Authoring/modifying the skill's internals (deferred unless interest grows)
- Using it to learn a real external topic yet
